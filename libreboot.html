libreboot

replacing the firmware they never wanted you to touch.

what is it

your BIOS is the first code your machine runs. before your OS, before your disk encryption, before anything you control — proprietary firmware is already executing. you can't read it. you can't audit it. you just trust it.

libreboot replaces that. it's a free, open-source firmware distribution built on top of coreboot, stripped of every proprietary binary blob possible. fully auditable. fully yours.

it boots fast, strips out surveillance infrastructure, and gives you a machine that actually belongs to you from the moment you press the power button.

intel ME — the computer inside your computer

Intel Management Engine is a separate processor embedded in every Intel chipset since 2008. it runs its own OS (based on Minix), has direct memory access, network access, and operates independently of your main CPU — even when the machine is "off."

you cannot disable it through software. you cannot inspect it. it persists through OS reinstalls. it predates your kernel loading. it has never been fully reverse engineered.

  • runs at ring -3 — below the kernel, below the hypervisor, below everything
  • has its own TCP/IP stack and can communicate over the network without OS involvement
  • Intel AMT (a feature of ME) allows remote access to the machine — screen, keyboard, storage
  • a critical vulnerability in AMT was disclosed in 2017 — silent remote auth bypass, existed since 2010
  • on older hardware (pre-ME v6), it can be completely removed
  • on modern hardware, libreboot uses me_cleaner to strip ~90% of ME firmware, leaving only what's needed to boot

on supported ThinkPads like the X200 and T400, libreboot removes ME entirely. gone. not disabled. removed.

the ME processor is on the chipset. you can't see it. that's intentional.
supported thinkpads

libreboot is selective. it only supports hardware where truly free firmware is achievable. most of its best-supported machines are ThinkPads — built to be repaired, documented, and opened.

ThinkPad X200 / X200s
ME fully removable. best choice for full freedom.
ThinkPad T400 / T400s
ME fully removable. GM45+ICH9M chipset.
ThinkPad X60 / X60s
no ME at all. oldest and cleanest supported machine.
ThinkPad T60
Intel GPU variants only. ATI models not supported.
ThinkPad X230
ME neutralized via me_cleaner. external flash required first.
ThinkPad T480 / T480s
newest supported. ME neutered via "deguard" technique. Dec 2024.

the T480 support is significant — it's the first modern ThinkPad to get libreboot, enabled by a newly discovered technique that breaks ME's security model entirely on that platform.

why bother
  • your current BIOS is closed source. nobody outside Intel and your OEM has audited it.
  • firmware-level malware survives OS reinstalls, disk wipes, everything
  • libreboot boots in ~1–2 seconds. proprietary BIOS takes 10–30.
  • no vendor lock-in. no phone-home. no update servers.
  • your threat model probably includes firmware if you're running cachy, mullvad, and tuta
  • it's the logical conclusion of caring about what runs on your machine
how to flash — step by step
a failed flash can permanently brick your machine. no recovery. no warranty. if you rush this, you will own a very flat paperweight. read everything twice before doing it once.

this guide covers the X230 (external flash) and X200 (internal flash possible). the T480 process differs — see libreboot.org for that specific board.

// EXTERNAL FLASHING (recommended for first flash)

01

what you need: a Raspberry Pi (any model), SOIC-8 clip, jumper wires, and the target ThinkPad. the clip attaches to the BIOS chip without soldering.

02

identify your BIOS chip. disassemble the ThinkPad and locate the SPI flash chip — usually labeled SST25VF016B or MX25L6406E. it's an 8-pin IC on the motherboard. check the libreboot docs for your specific board's chip location.

03

wire the clip to the Pi. connect the SOIC clip to the Pi's GPIO pins. standard wiring: VCC → 3.3V, GND → GND, CS → CE0, CLK → SCLK, MOSI → MOSI, MISO → MISO. do not power the ThinkPad while clipped.

04

install flashprog on the Pi and verify you can read the chip:

# install flashprog
sudo apt install flashprog
# read existing BIOS — do this twice and diff the outputs
sudo flashprog -p linux_spi:dev=/dev/spidev0.0,spispeed=512 -r backup1.bin
sudo flashprog -p linux_spi:dev=/dev/spidev0.0,spispeed=512 -r backup2.bin
diff backup1.bin backup2.bin
# if diff is silent, your read is clean. keep backup1.bin somewhere safe.
05

download the libreboot ROM for your board from libreboot.org. verify the SHA512 checksum. pick the right payload — SeaGRUB if you want GRUB in flash, SeaBIOS if you want legacy boot.

06

insert vendor files if using a release ROM (not source build). libreboot release ROMs ship without certain vendor-specific files. run lbmk to inject them from your backup:

./vendor inject -r libreboot.rom -b x230_12mb
07

flash it:

sudo flashprog -p linux_spi:dev=/dev/spidev0.0,spispeed=512 \
  -w libreboot.rom --verify
# --verify re-reads after writing and compares. non-negotiable.
08

remove the clip, reassemble, power on. if you see GRUB or SeaBIOS — you're done. if you see nothing, stay calm, re-clip, and re-flash from your backup.

// INTERNAL FLASHING (X200, X60, some others)

on machines like the X200 where internal flashing is possible from Lenovo BIOS, libreboot ships two patched flashrom binaries to handle different BIOS chip manufacturers:

# run both — only one will succeed depending on your chip
sudo ./flashrom_lenovobios_sst -p internal -r backup.bin
sudo ./flashrom_lenovobios_macronix -p internal -r backup.bin
# whichever created backup.bin — use that binary to flash
sudo ./flashrom_lenovobios_[sst|macronix] -p internal -w libreboot.rom

bricking risks
brick = the machine no longer boots. no OS, no BIOS screen, nothing. the only recovery is re-flashing externally — which requires another working machine and the hardware clip setup. if you didn't back up your original BIOS, recovery may be impossible.
  • flashing the wrong ROM for your board — verify model and chip size (8MB vs 12MB)
  • dirty read — if your two backup reads don't match, the connection is unstable. fix before writing.
  • power loss mid-flash — keep everything on stable power. no laptops running on battery.
  • flashing a T400/T60 with an ATI GPU variant — only Intel GPU models are supported
  • skipping vendor file injection on release ROMs — machine won't initialize memory properly
  • Boot Guard — on Haswell (4th gen Intel) and newer OEM machines, Intel Boot Guard cryptographically prevents unsigned firmware. libreboot cannot run on these. check your hardware before buying.

the T480 is specifically supported because Lenovo did not enable Boot Guard on it. this is rare. most modern ThinkPads are locked.

libreboot vs the alternatives
firmware blob-free ME removed ease of use hardware support
libreboot yes yes / partial moderate limited
coreboot no (blobs allowed) partial hard wide
Heads mostly partial very hard very limited
osboot mostly partial moderate moderate
vendor BIOS no no easy full

coreboot is the foundation everything else builds on — including libreboot. but coreboot itself allows vendor blobs (Intel FSP, AMD AGESA) for hardware support. it's open source but not free. the hardware compatibility is broader but the firmware isn't clean.

Heads is interesting — it's designed for measured boot and tamper detection, pairs with a TPM, and is used by high-security setups like Purism and some Qubes configurations. technically impressive. extremely difficult. not designed for general use.

osboot is essentially libreboot's sibling — also a coreboot distribution, also focused on freedom, slightly more pragmatic about blob inclusion on some boards. less strict than libreboot.

libreboot is the most principled of the group. it refuses blobs even when that means dropping hardware support. that inflexibility is a feature, not a bug. if it's on the supported list, it's actually clean. no asterisks.

where to start
  • if you have an X200 or T400 — internal flash is possible. cleanest result. ME fully gone.
  • if you have an X230 — external flash required for first install. worth doing.
  • if you want modern hardware — T480/T480s are your best option right now. also available pre-flashed from certain vendors.
  • official docs: libreboot.org/docs/install
  • don't have compatible hardware — buy pre-flashed. the process is documented, the risk is real.
