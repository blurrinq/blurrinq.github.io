lightweight linux, ranked

opinionated. what actually runs well on old iron in 2026.

before we start

this isn't a "best distros for beginners" list. this is for people who want to give a 10–20 year old ThinkPad a second life and understand that the choice of distro matters as much as the hardware.

the rule: the lighter the distro, the more it asks of you. you can't have zero RAM usage and a hand-holding installer. pick your trade-off.

all RAM figures are approximate idle usage with a minimal desktop or none.

// S tier
Void Linux
rolling · runit · independent
RAM (idle)
~80–120MB
install difficulty
moderate
daily driver?
yes

void is the answer most people end up at after trying everything else. it uses runit instead of systemd — meaning it starts faster, uses less memory, and has no hidden daemons running things you didn't ask for. it has its own package manager (XBPS) that's fast and doesn't get in the way.

two variants: glibc (broader software compatibility) and musl (smaller, faster, some software won't compile). on a ThinkPad, use glibc unless you know exactly why you want musl.

  • no systemd. runit is simpler and faster.
  • rolling release — always current, no version upgrades
  • XFCE, i3, sway, dwm — pick your DE/WM at install or after
  • active development, good package coverage
  • the documentation is honest. it doesn't pretend things are easier than they are.
best for: people who want a clean, fast, fully controllable daily driver on older hardware. the sweet spot between usability and minimalism.
// S tier
CachyOS
rolling · systemd · Arch-based
RAM (idle)
~300–500MB
install difficulty
easy
daily driver?
yes

not ultralight, but extremely fast for what it is. CachyOS ships with a custom kernel (BORE scheduler, optimized builds) that makes it noticeably snappier than vanilla Arch on the same hardware. if you're on a ThinkPad T480 or anything post-2015, this is where I'd start.

heavier than void or alpine but still far leaner than Ubuntu or Fedora. the AUR access is the practical win — nearly any software you need is packaged somewhere.

  • custom kernel with performance patches — makes a real difference
  • x86-64-v3 optimized builds for capable CPUs
  • good hardware support out of the box — ThinkPads just work
  • Arch base means rolling, up-to-date, and sometimes breaks things
best for: modern-ish ThinkPads (X230 and newer) where you want speed and convenience without bloat. also: what I run.
// A tier
Alpine Linux
stable · OpenRC · musl · independent
RAM (idle)
~30–50MB
install difficulty
hard
daily driver?
possible

alpine uses musl libc and busybox instead of glibc and GNU coreutils. it is tiny. the base install is under 200MB. it boots in seconds and idles at almost nothing.

the trade-off: musl breaks some software. things compiled against glibc need to be recompiled or run in a compatibility layer. if you're doing anything beyond a terminal environment, expect to solve problems. the docker world loves alpine. desktop use is possible but requires effort.

  • genuinely tiny. no fat anywhere.
  • OpenRC init — fast, clean, understandable
  • strong security defaults — hardened kernel configs available
  • some packages missing or broken due to musl incompatibility
  • best for servers, containers, and people who know what they're doing
best for: very old hardware (pre-2008), containers, servers, or if you want to understand linux from the ground up. not for beginners.
// A tier
Artix Linux
rolling · runit/OpenRC/s6 · Arch-based
RAM (idle)
~120–200MB
install difficulty
moderate
daily driver?
yes

arch linux without systemd. you get the AUR, the rolling release model, the package coverage — without the init system that half the linux world is tired of arguing about. choose runit, OpenRC, or s6 depending on preference.

  • AUR access — massive software availability
  • systemd-free — runs meaningfully leaner
  • familiar to anyone who knows Arch
  • slightly smaller community than Arch itself
best for: Arch users who want to drop systemd without learning a new ecosystem from scratch.
// B tier
antiX
stable · sysvinit · Debian-based
RAM (idle)
~100–180MB
install difficulty
easy
daily driver?
yes

Debian-based but systemd-free, designed explicitly for old hardware. it ships with lightweight window managers (IceWM, fluxbox) and a full desktop out of the box. the most accessible option on this list — you can hand it to someone who's never used linux and they'll be okay.

the trade-off is the Debian base — stable, yes, but packages lag behind. you won't get the latest software versions quickly.

  • 32-bit support — good for truly ancient hardware
  • no systemd, ships functional out of the box
  • Debian's stability and package ecosystem
  • conservative release cycle — not rolling
best for: very old ThinkPads (T60, X60 era), people new to linux, anyone who needs a working desktop on minimal hardware with minimal setup.
// specialist
Tiny Core Linux
modular · BusyBox · independent
RAM (idle)
~46MB minimum
install difficulty
very hard
daily driver?
unlikely

tinycore is a category of its own. the base image is 24MB. it runs entirely in RAM. it requires just 46MB of RAM to boot. it keeps machines from the early 2000s alive — machines with Pentium 4 processors and 256MB of RAM that no other distro will touch.

the architecture is modular: you boot a minimal core and load only the extensions you need. everything is ephemeral by default. nothing is installed — it's loaded into RAM on boot from a local directory or the internet. the concept is almost alien the first time you encounter it.

the trade-offs are severe. tinycore is not a general-purpose OS. it is a survival tool for hardware that should be dead. it requires understanding the extension model, manual package management, and comfort with a system that will forget changes on reboot unless explicitly persisted.
  • 24MB ISO (Core), 248MB with WiFi support (CorePlus)
  • i486DX minimum CPU. genuinely runs on 20-year-old machines.
  • runs entirely from RAM — fast on slow hardware
  • WiFi: use CorePlus, or be on a wired connection
  • software: extension-based. the repository exists but it's limited.
  • persistence requires manual setup — nothing survives a reboot by default
  • documentation is sparse and assumes knowledge
best for: resurrecting hardware from 2000–2008 that otherwise goes to landfill. also: a fascinating experiment in how minimal a working OS can actually be. not for primary use.
the honest summary

you are not going to get 2026 performance from 2006 hardware regardless of the distro. the CPU is what it is. the RAM ceiling is real. video conferencing at 1080p is going to stutter on anything pre-2012.

what you can get:

  • a fast, responsive terminal environment on hardware that costs almost nothing
  • a browser session that doesn't lag — on void or alpine with a minimal WM
  • a programming environment that feels snappy — LSP, compiler, git — all fine
  • a machine you fully understand and fully control, down to the firmware

the distro is only part of the equation. pair it with a minimal window manager (i3, sway, dwm, openbox) instead of a full DE, kill the services you don't use, and a 10-year-old ThinkPad feels like a tool again instead of a liability.

the machine isn't slow. the software you've been running on it is.
quick reference
  • T480/X230 and newer — CachyOS or Void
  • X220/T420 era — Void or Artix
  • X200/T400 era — Alpine or antiX
  • pre-2006 hardware — TinyCore, antiX 32-bit
  • servers / containers — Alpine, always Alpine
