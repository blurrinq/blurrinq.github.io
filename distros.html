lightweight linux, ranked

opinionated. what actually runs well on old iron in 2026.

before we start

this isn't a "best distros for beginners" list. this is for people who want to give a 10–20 year old ThinkPad a second life and understand that the choice of distro matters as much as the hardware.

the rule: the lighter the distro, the more it asks of you. you can't have near-zero RAM usage and a hand-holding installer. pick your trade-off. all RAM figures are approximate idle usage with a minimal desktop or none.

// S tier
Void Linux
rolling · runit · independent
void + i3wm — minimal by default, everything else is your choice
RAM (idle)
~80–120MB
difficulty
moderate
daily driver?
yes

void is the answer most people end up at after trying everything else. runit instead of systemd — starts faster, less memory, no hidden daemons. XBPS package manager is fast and doesn't get in the way.

two variants: glibc (broader compatibility) and musl (smaller, faster, some software won't compile). on a ThinkPad, use glibc unless you know exactly why you want musl.

  • no systemd. runit is simpler and faster at boot.
  • rolling release — always current, no version upgrade headaches
  • pick your WM: i3, sway, dwm, XFCE — all work great
  • documentation is honest. doesn't pretend things are easier than they are.
best for: people who want a clean, fast, fully controllable daily driver on older hardware. the sweet spot between usability and minimalism.
// S tier
CachyOS
rolling · systemd · Arch-based
cachyos + KDE — arch with a custom kernel that actually makes a difference
RAM (idle)
~300–500MB
difficulty
easy
daily driver?
yes

not ultralight, but extremely fast for what it is. ships with a custom kernel (BORE scheduler, optimized builds) that makes it noticeably snappier than vanilla Arch on the same hardware. ThinkPads work out of the box.

  • custom kernel with BORE scheduler — real difference on older CPUs
  • x86-64-v3 optimized builds on capable hardware
  • full AUR access — nearly any software you need is packaged somewhere
  • rolling — up to date, occasionally breaks. know how to recover.
best for: X230 and newer where you want speed and AUR without bloat. also what I run.
// A tier
Alpine Linux
stable · OpenRC · musl · independent
alpine + dwm — if it's on screen, you put it there deliberately
RAM (idle)
~30–50MB
difficulty
hard
daily driver?
possible

alpine uses musl libc and busybox. base install under 200MB, idles at almost nothing. the docker world runs on it for a reason. desktop use is possible but requires patience.

  • OpenRC init — fast, clean, easy to read and understand
  • strong security defaults, hardened kernel configs available
  • musl breaks some glibc software — expect to troubleshoot
  • excellent for servers, containers, or very RAM-constrained machines
best for: pre-2008 hardware, containers and servers, or if you want to understand linux properly from the ground up.
// A tier
Artix Linux
rolling · runit/OpenRC/s6 · Arch-based
artix + openbox — arch without the systemd debate
RAM (idle)
~120–200MB
difficulty
moderate
daily driver?
yes

Arch without systemd. full AUR access, rolling release, the full package coverage — without the init system. choose runit, OpenRC, or s6 at install.

  • AUR access — massive software availability
  • systemd-free — leaner and faster than vanilla Arch
  • familiar to any Arch user
  • slightly smaller community than Arch proper
best for: Arch users who want to drop systemd without learning a new ecosystem from scratch.
// B tier
antiX
stable · sysvinit · Debian-based
antiX + IceWM — a full working desktop under 180MB RAM
RAM (idle)
~100–180MB
difficulty
easy
daily driver?
yes

Debian-based, systemd-free, built for old hardware. ships with IceWM and fluxbox out of the box. the most accessible option on this list — hand it to someone who's never used linux and they'll be okay.

  • 32-bit support — good for truly ancient hardware
  • no systemd, ships functional immediately
  • Debian's stability — packages are tested and reliable
  • conservative release cycle — packages lag a little behind current
best for: T60/X60 era machines, anyone new to linux on old hardware, or anyone who needs a working desktop with minimal setup time.
// specialist
Tiny Core Linux
modular · BusyBox · independent
tinycore — 24MB ISO. runs entirely in RAM. the whole OS.
RAM (idle)
~46MB total
difficulty
very hard
daily driver?
unlikely

the base image is 24MB and runs entirely in RAM. keeps machines from the early 2000s alive — Pentium 4, 256MB RAM, hardware no other distro will touch. the architecture is modular: boot a minimal core, load only the extensions you need.

the trade-offs are severe. tinycore is not a general-purpose OS. it's a survival tool for hardware that should be landfill. nothing persists across reboots by default. WiFi, desktop, and persistence all require manual configuration. not for beginners.
  • 24MB ISO (Core) — 248MB with GUI and WiFi (CorePlus)
  • i486DX minimum. genuinely runs on 20+ year old machines.
  • entirely RAM-based — fast even on slow or dying storage
  • extension repo is limited. expect to compile things yourself.
best for: resurrecting pre-2005 hardware that would otherwise go to landfill. also a fascinating study in how minimal a working OS can actually be.
the honest summary

you are not getting 2026 performance from 2006 hardware regardless of the distro. the CPU is what it is. video calls at 1080p will stutter on anything pre-2012. that's just physics.

what you can get: a fast terminal, a usable browser, a full dev environment, and a machine you fully understand. pair any of these with a minimal WM — i3, sway, dwm, openbox — and old hardware feels like a proper tool again.

the machine isn't slow. the software you've been running on it is.
  • T480 / X230 and newer — CachyOS or Void
  • X220 / T420 era — Void or Artix
  • X200 / T400 era — Alpine or antiX
  • pre-2006 hardware — TinyCore or antiX 32-bit
  • servers / containers — Alpine, always
